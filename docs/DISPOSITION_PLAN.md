# Kira Disposition Plan

This document assigns every major module or script to one of five scientific dispositions:

- `move to Evidence`
- `keep in Engine`
- `move to Hypothesis`
- `split`
- `deprecate`

The reasoning below is scientific, not software-maintenance-oriented.

## Move to Hypothesis

| Module | Disposition | Scientific reason |
|---|---|---|
| `src/kira/causality/__init__.py` | move to Hypothesis | The implemented science is not causality; it is mechanistic hypothesis generation. |
| `src/kira/causality/binding_site.py` | move to Hypothesis | It defines centroid- or coordinate-based pockets and residue comparisons that suggest where selectivity-relevant differences might live. |
| `src/kira/causality/divergence.py` | move to Hypothesis | It generates local physicochemical divergence profiles that are biologically suggestive, not validated mechanism. |
| `src/kira/causality/energy_decomp.py` | move to Hypothesis | It yields residue-energy heuristics from protein-only LJ decomposition. Those are interpretive hypotheses, not ligand-binding facts. |
| `src/kira/causality/selectivity_map.py` | move to Hypothesis | It assembles heuristic residue attributions that can suggest mechanistic stories but do not establish them. |
| `src/kira/physics/core/energy.py` | move to Hypothesis | Its residue-level energy outputs are used interpretively and are too approximate to remain purely an engine-level physical truth layer. |

## Split

| Module | Disposition | Scientific reason |
|---|---|---|
| `src/kira/data/target_manifest.py` | split | It mixes curated biological ontology with internal validation of historical dataset consistency. Comparator truth and dataset agreement are distinct scientific objects. |
| `src/kira/cli.py` | split | One CLI currently spans curation lookup, benchmark scoring, approximate structure QC, and mechanistic hypothesis output. These belong to Evidence, Engine, and Hypothesis respectively and should not share one undifferentiated scientific voice. |
| `src/kira/physics/config.py` | split | It parameterizes both implemented checks and unimplemented aspirational checks. Active thresholds and planned-check thresholds are different scientific categories. |
| `tests/test_selectivity_v3.py` | split | It mixes neutral feature-regression checks with tests that lock in the project’s own biological hypothesis as if it were stable truth. |
| `tests/test_target_manifest.py` | split | It mixes reproducibility checks with tests that canonize biological comparator choices. Internal consistency and biological validity are distinct. |

## Deprecate

| Module | Disposition | Scientific reason |
|---|---|---|
| `src/kira/dock/__init__.py` | deprecate | There is no active docking science here. Keeping it visible implies a live structural selectivity capability the active package does not have. |
| `src/kira/eval/__init__.py` | deprecate | It suggests an active evaluation layer that does not exist as a meaningful scientific subsystem. |
| `src/kira/filter/__init__.py` | deprecate | It implies a live filtering science layer without actual content. |
| `src/kira/graph/__init__.py` | deprecate | It implies active graph-based biomedical reasoning, but that work is historical and archived. |
| `archived/scripts/22_build_target_manifest.py` | deprecate | It adds no independent scientific content beyond wrapping the active manifest pipeline. |
| `tests/__init__.py` | deprecate | It has no scientific meaning on its own and should not appear in scientific doctrine. |

## Scientific Summary

### Evidence

Use this artifact for:

- curated priors
- historical target mappings
- historical translational rankings
- archived benchmark outcomes
- archived docking findings
- manuscript-era narratives

### Engine

Use this artifact for:

- approximate structural representation and QC
- standard chemistry heuristics
- pair-level pocket-feature computation
- small retrospective benchmarking

### Hypothesis

Use this artifact for:

- residue-level divergence interpretation
- approximate energetic storytelling
- proposed mechanistic rationales
- selectivity hypotheses to be tested elsewhere

### Deprecation Principle

If a module implies active docking, active graph science, active evaluation science, or active filter science without real scientific content, it should remain outside the active claims surface.
